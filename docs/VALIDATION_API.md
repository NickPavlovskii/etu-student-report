# API проверки документов: коды критериев и иллюстрации/таблицы

Документ описывает коды критериев в ответе валидации и раздел «Иллюстрации и таблицы» для фронтенда.

---

## 1. Таблица критериев — коды

В ответе `ValidationResult.criteria` каждый элемент содержит поле `code`. Ниже — полный список кодов.

| code | title (пример) | Описание |
|------|----------------|----------|
| FILE_FORMAT | Формат файла | .doc, .docx, .pdf |
| FONT | Шрифт | Times New Roman и т.п. |
| FONT_SIZE | Размер шрифта | 14 пт |
| LINE_SPACING | Межстрочный интервал | 1.5 |
| MIN_PAGES | Минимальное количество страниц | Кол-во страниц |
| MIN_SOURCES | Минимальное количество источников | Кол-во в списке литературы |
| HAS_TOC | Оглавление | Наличие «содержание»/«оглавление» |
| HAS_INTRODUCTION | Введение | Наличие «введение» |
| HAS_CONCLUSION | Заключение | Наличие «заключение» |
| HAS_BIBLIOGRAPHY | Список источников/литературы | Наличие раздела |
| HAS_APPENDICES | Приложения | Наличие «приложени» |
| HAS_TITLE_PAGE | Титульный лист по шаблону | Обязательные элементы из titlePageRequiredStrings |
| **ILL_NUMBERING** | Нумерация иллюстраций | Сквозная нумерация (если в шаблоне задано `illNumbering`) |
| **FIGURE_POSITION** | Позиция рисунка | По центру страницы, только DOCX (если в шаблоне задано `figurePosition`) |
| **FIGURE_CAPTION_POS** | Подпись рисунка | Под рисунком (если в шаблоне задано `figureCaption`) |
| **TABLE_TITLE** | Заголовок таблицы | «Таблица» + номер (если в шаблоне задано `tableTitle`) |
| **TABLE_POSITION** | Позиция таблицы | По центру страницы / слева и т.д. (если в шаблоне задано `tablePosition`) |

## 2. Иллюстрации и таблицы (для фронтенда)

### 2.1. Когда критерии попадают в ответ

Коды **ILL_NUMBERING**, **FIGURE_POSITION**, **FIGURE_CAPTION_POS**, **TABLE_TITLE** возвращаются в `criteria` **только если в шаблоне заданы соответствующие поля**:

| Поле шаблона (criteria) | Код в ответе |
|-------------------------|---------------|
| `illNumbering`          | ILL_NUMBERING |
| `figurePosition`        | FIGURE_POSITION |
| `figureCaption`         | FIGURE_CAPTION_POS |
| `tableTitle`            | TABLE_TITLE |
| `tablePosition`         | TABLE_POSITION |

Если в шаблоне нет, например, `figurePosition`, то критерий FIGURE_POSITION в ответе не появится.

### 2.2. Что проверяется по каждому полю

- **ILL_NUMBERING** — нумерация иллюстраций (сквозная): соответствие заданному в шаблоне значению (например, «Сквозная»).
- **FIGURE_POSITION** — позиция рисунка (например, «По центру страницы»). Проверяется **только для DOCX**. Для PDF этот критерий не возвращается и не проверяется.
- **FIGURE_CAPTION_POS** — подпись рисунка (например, «Под рисунком»).
- **TABLE_TITLE** — заголовок таблицы в виде «Таблица» + номер (согласно шаблону).
- **TABLE_POSITION** — позиция таблицы на странице (по центру, слева и т.д.).

Проверка выполняется по триггерным словам в документе: «Рис.», «Рисунок», «Таблица» и т.п.

### 2.3. Уровень при нарушении

При нарушении критериев по иллюстрациям и таблицам бэкенд возвращает **уровень `warning`** (не `error`). То есть в `CriteriaResult` поле `level` будет `"warning"`.

### 2.4. Отображение в таблице результатов

Эти критерии отображаются **как и остальные**: по одному ряду на критерий с полями:

- `title` — название критерия;
- `passed` — пройден / не пройден;
- `level` — `"error"` или `"warning"`;
- `expected` — ожидаемое значение (из шаблона);
- `actual` — фактическое значение (из документа).

Фронт может использовать общую таблицу результатов по списку `criteria` без отдельной логики для иллюстраций и таблиц.

### 2.5. Кратко

- Проверка идёт по триггерным словам («Рис.», «Рисунок», «Таблица»).
- Для **PDF** критерий **FIGURE_POSITION** не проверяется и в ответе не приходит (если в шаблоне задан `figurePosition`, бэкенд для PDF его не добавляет в результат).
- Уровень при нарушении — **warning**.
