# Интеграция фронтенда с API проверки документов

Краткое руководство для фронтенда по использованию API валидации и шаблонов.

---

## 1. Основные документы

- **API_VALIDATION.md** — описание API (эндпоинты, форматы запросов/ответов, TypeScript-интерфейсы), список кодов критериев и сценарии интеграции.
- **VALIDATION_API.md** — таблица кодов критериев (включая ILL_NUMBERING, FIGURE_POSITION, FIGURE_CAPTION_POS, TABLE_TITLE) и раздел **«Иллюстрации и таблицы»** (когда эти критерии попадают в ответ, соответствие полей шаблона и кодов, что проверяется, как отображать в таблице результатов).

Фронт может опираться на **VALIDATION_API.md** для:
- полного списка кодов критериев в ответе валидации;
- раздела про иллюстрации и таблицы (условия появления критериев, уровень warning, отображение).

---

## 2. Поля шаблона «Иллюстрации и таблицы»

Поля шаблона проверки **illNumbering**, **figurePosition**, **figureCaption**, **tableTitle** проверяются **бэкендом**. Они соответствуют кодам в ответе валидации и правилам, описанным в **VALIDATION_API.md** → раздел **«Иллюстрации и таблицы»**.

На фронте достаточно:
- передавать эти поля в шаблоне при создании/редактировании (если форма их поддерживает);
- отображать в таблице результатов критерии с кодами ILL_NUMBERING, FIGURE_POSITION, FIGURE_CAPTION_POS, TABLE_TITLE так же, как остальные критерии (title, passed, level, expected, actual).

Детали логики проверки (триггерные слова, только DOCX для FIGURE_POSITION, уровень warning) см. в **VALIDATION_API.md**.
