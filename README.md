# ИС «Отчёты»

**Веб-клиент информационной системы учёта учебных работ** для кафедры информатики и прикладной математики **[СПбГЭТУ «ЛЭТИ»](https://etu.ru/)** — федерального автономного университета (основан в **1886** году).

*Vue 3 · TypeScript · Vite*

</div>

---

## О системе

**Информационная система (ИС)** предназначена для автоматизации работы с учебными работами в вузе:

| Роль / сценарий | Возможности |
|-----------------|-------------|
| **Преподаватель** | Просмотр своих дисциплин, загрузка и проверка отчётов студентов, переход в архив по работам |
| **Кафедра / администратор** | Управление пользователями, просмотр дисциплин кафедры, журнал событий (аудит) |
| **Аналитика** | Сводные данные по ошибкам и показателям (модуль аналитики) |
| **Настройки** | Типы контроля и связанные параметры для дисциплин |

Этот репозиторий — **фронтенд (SPA/PWA)**: интерфейс в браузере, данные приходят с **бэкенд-API** (отдельный сервер).

## Бизнес-процесс

Сквозной процесс работы с шаблонами проверки, привязкой к дисциплинам и загрузкой работ с автопроверкой.

<div align="center">

<img src="docs/diagrams/business-process-bpmn.png" alt="BPMN: бизнес-процесс ИС «Отчёты» — авторизация, шаблоны, загрузка, автопроверка, хранение" width="920" />

<p><em>Модель процесса в нотации BPMN 2.0 (Camunda): ETU ID → шаблоны и виды контроля → одиночная / массовая загрузка (файлы или Moodle) → <code>POST /validate</code> → <code>POST /reports</code> → хранилище отчётов.</em></p>

### Фаза 1. Создание шаблонов проверки

| Шаг | Действие | Результат |
|-----|----------|-----------|
| 1 | Авторизация в системе | Пользователь вошёл в ИС |
| 2 | Открытие раздела «Настройки» | Вкладка «Доступные шаблоны» |
| 3 | Создание шаблона: форматирование, объём | Заполнены критерии оформления |
| 4 | Настройка критериев иллюстраций и таблиц | Полный набор правил проверки |
| 5 | Сохранение | Шаблон в БД (`verification_templates` / `POST /api/templates`) |

### Фаза 2. Привязка шаблонов к видам контроля

| Шаг | Действие | Результат |
|-----|----------|-----------|
| 1 | Выбор дисциплины из списка | Выбран `planRowId` |
| 2 | Активация видов контроля (toggle) | Включены ИДЗ, ИДРЗ, курсовая и т.д. |
| 3 | Выбор шаблона и тем для каждого вида | Связь «вид контроля → шаблон» |
| 4 | Сохранение | Конфигурация (`discipline_control_templates` / localStorage на клиенте) |

### Фаза 3. Загрузка работ с автопроверкой

| Шаг | Действие | Результат |
|-----|----------|-----------|
| 1 | Открытие страницы дисциплины | Таблица групп и студентов |
| 2 | Загрузка работы (файл + метаданные) | Одиночная или массовая загрузка |
| 3 | Автопроверка по шаблону | `POST /api/validate` по `templateId` |
| 4 | Отчёт: процент соответствия, ошибки | Преподаватель подтверждает загрузку |
| 5 | Сохранение работы | Отчёт в БД (`student_reports` / `POST .../reports`) |

**Роли:** преподаватель (фазы 1–3), администратор кафедры и зав. кафедры — те же шаги при ведении дисциплин + управление кафедрой.

## Быстрый старт

### Требования

- **Node.js** 18+ (рекомендуется актуальный LTS)
- **npm** (или `pnpm` / `yarn`)

### Установка зависимостей

```bash
npm install
```

### Режим разработки

```bash
npm run dev
```

Откройте в браузере адрес из консоли (обычно **http://localhost:5173**).  
Для работы с данными нужен **бэкенд**: в dev-режиме запросы к `/api` проксируются на **http://localhost:8081** (см. `vite.config.ts`).

### Продакшен-сборка

```bash
npm run build
```

Выполняется проверка типов (`vue-tsc`) и сборка. Результат:

`dist/frontend/contents/panda/client`

Путь задан под раздачу статики бэкендом; при необходимости измените `build.outDir` в `vite.config.ts`.

### Тесты

```bash
npm run test
```

Режим разработки с автоперезапуском: `npm run test:watch`. Подробности — в разделе [Тесты и Storybook](#тесты-и-storybook).

### Storybook

```bash
npm run storybook
```

Интерфейс откроется на **http://localhost:6006**.
